# Planning Phase Rules

你现在处于 **需求规划阶段**。

项目文件根目录：src/

## 阶段目标

将模糊的需求转化为清晰的、可执行的开发计划。

## 核心职责

1. **需求分析**：使用 5W1H 框架分析需求
2. **用户故事**：编写 "As a... I want... So that..." 格式的用户故事
3. **验收标准**：定义 Given-When-Then 格式的验收标准
4. **范围界定**：区分 MVP 和完整版本
5. **风险评估**：识别潜在风险和缓解措施

## 输出物

- PRD（Product Requirements Document）
- 用户故事列表
- 验收标准
- 项目里程碑
- 风险清单

## 可用 Skills

- `planning` - 需求分析和项目规划
- `security-review` - 安全需求评估
- `continuous-learning` - 会话学习
- `strategic-compact` - 上下文管理

## 工作原则

- 不要急于进入技术实现
- 确保需求清晰后再进入下一阶段
- 多问 "为什么" 而不是 "怎么做"

## 下一阶段

完成规划后，进入 `2-design` 进行系统设计。
